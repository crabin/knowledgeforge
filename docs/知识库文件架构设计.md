下面给你一套**通用完整知识领域目录结构模板**。  
它可以用于任何领域，比如：

```text
Deep Learning
Machine Learning
Computer Vision
Cybersecurity
AI Agent
Backend Development
Natural Language Processing
```

核心原则是：

> `00_overview` 负责建立整体认知；  
> `01_foundations` 负责补基础；  
> `02_core_topics` 负责学习主干知识；  
> `03_advanced_topics` 负责深入和前沿；  
> `04_papers` 负责论文阅读；  
> `05_projects` 负责实践输出；  
> `06_tools` 负责工具框架；  
> `07_review` 负责复习、总结和面试。

# 二、根目录文件说明

## 1. `index.md`

作用：整个知识库的**全局索引文件**。

它应该包含：

```text
1. 知识库整体结构
2. 每个目录的作用
3. 学习顺序
4. 当前学习进度
5. 重要文件链接
6. 推荐阅读顺序
```

推荐结构：

```markdown
# Knowledge Index: [Field Name]

## 1. Purpose

这个知识库用于系统学习和整理 [Field Name] 相关知识。

## 2. Global Structure

| Directory | Purpose |
|---|---|
| 00_overview | 领域背景 |
| 01_foundations | 基础知识 |
| 02_core_topics | 核心分支 |
| 03_advanced_topics | 高级方向 |
| 04_papers | 论文阅读 |
| 05_projects | 项目实践 |
| 06_tools | 工具框架 |
| 07_review | 复习总结 |

## 3. Recommended Learning Order

1. README.md
2. 00_overview/
3. 01_foundations/
4. 02_core_topics/
5. 03_advanced_topics/
6. 04_papers/
7. 05_projects/
8. 07_review/

## 4. Learning Progress

| Stage | Status | Notes |
|---|---|---|
| Overview | Not Started | - |
| Foundations | Not Started | - |
| Core Topics | Not Started | - |
| Advanced Topics | Not Started | - |
| Papers | Not Started | - |
| Projects | Not Started | - |
| Review | Not Started | - |
```

## 2. `README.md`

作用：领域的**总览说明文件**。

它不是目录，而是帮助学习者快速理解：

```text
这个领域是什么？
为什么重要？
发展历史是什么？
现在研究到哪里了？
可以用于哪些场景？
应该怎么学？
```

推荐结构：

```markdown
# [Field Name] Overview

## 1. What is [Field Name]?

简要定义这个领域。

## 2. Why is it Important?

说明这个领域的重要性。

## 3. Core Problems

| Problem | Description |
|---|---|
| Problem 1 | 说明 |
| Problem 2 | 说明 |

## 4. Development History

按阶段说明发展过程。

## 5. Current Research Status

说明当前主流研究方向。

## 6. Application Areas

说明应用场景。

## 7. Learning Path

说明推荐学习路线。

## 8. Expected Output

学习完成后应该产出：

- 知识笔记
- 论文总结
- 代码项目
- 实验报告
- 研究问题
```
